# v0.1.14 Accessibility Audit

**Date:** 2026-06-13  
**Scope:** Authenticated navigation, campaign onboarding, form builder, analytics, global search, and public feedback form.

## Checks

- Keyboard operation and focus return for the global search dialog.
- Current-page announcement through `aria-current`.
- Visible text retained for icon-enhanced actions.
- Accessible table alternative for analytics charts.
- Labels, headings, form controls, and public-form honeypot exposure.
- Desktop and narrow mobile layouts, including horizontal page overflow.
- Light and dark themes, visible focus, touch-target sizing, and reduced-motion behavior.
- English, German, and Spanish catalog completeness through automated tests.

## Fixes

- Added explicit current navigation states.
- Added keyboard shortcuts, Escape handling, and trigger focus restoration for global search.
- Hid decorative SVG charts and icons from assistive technology while preserving text and tabular data.
- Removed the public honeypot from the accessibility tree.
- Increased mobile control targets and prevented tooltip-generated viewport overflow.
- Added reduced-motion overrides and clearer loading and unsaved-change states.

## Result

No release-blocking accessibility defect was found after the fixes. Automated Go/template/i18n tests and manual browser checks cover the changed surfaces.

## Remaining Limitations

- This review is not a formal WCAG conformance certification.
- Native horizontal scrolling remains intentional for wide analytics tables and campaign subnavigation on narrow screens.
- Operator-provided campaign wording and color-customization changes require their own accessibility review.
