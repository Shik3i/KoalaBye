# KoalaBye v0.1.14

Released 2026-06-13.

## Analytics

- privacy-preserving visit-to-form-start-to-submission funnel
- 7, 30, and 90 day previous-period comparisons
- filters for dates and collected app, extension, platform, browser, and operating-system values
- authorized same-organization campaign comparison
- field-value trends for ratings, radio choices, and checkbox choices
- filter-aware CSV and JSON exports with configurable CSV columns and formula neutralization

## Workflow

- campaign list search, status and redirect filters, and sorting by actual visit or submission activity
- global keyboard-accessible campaign search with direct section links
- campaign onboarding checklist covering privacy, form setup, preview, activation, and a real test response
- active navigation states, unsaved-change warnings, and loading states

## Accessibility

- local inline action icons retain visible labels
- improved keyboard dialog behavior and focus return
- chart data has an accessible table alternative
- wider touch targets, reduced-motion support, and mobile overflow fixes
- release review recorded in `docs/audits/V0.1.14_ACCESSIBILITY_AUDIT.md`

## Privacy

A form start is recorded only after the first interaction, once per existing anonymous visit. It stores no field identity, answer, draft, cookie, IP address, raw user agent, or additional visitor identifier.

Diagnostic filters and export columns are available only for collection enabled by the campaign owner. Public forms continue to work without mandatory JavaScript; failure to send the optional form-start event does not block submission.

## Upgrade Notes

Migration `00012_analytics_funnel.sql` adds the form-start table and indexes. It is forward-only and runs automatically. Back up the SQLite database before upgrading.

Retention remains manual, deployment-specific legal review remains the operator's responsibility, and releases below v1.0.0 remain experimental.
